💡 关键创新：GPT-OSS使用了Mixture of Experts (MoE)架构，虽然总参数量达到117B，但每次推理只激活5.1B参数，大幅提升了效率。同时采用4位量化（MXFP4）技术，使模型可以在单个H100 GPU上运行。

🏗️ Transformer 架构层级

📥 输入层 (Input Embedding)

词嵌入 + 位置编码（RoPE）

Token Embedding (4096维) + Rotary Position (128K上下文)
🧠 Transformer层 x 80层

交替使用密集注意力和局部带状稀疏注意力

多头注意力（96头）：
Head 1 Head 2 Head 3 ... Head 96
MoE前馈网络（32个专家）：
Expert 1 ✓ Expert 2 Expert 3 ✓ Expert 4 ... Expert 32

每个token选择Top-2专家进行处理

📤 输出层

投影到词汇表空间（200K词汇）

# GPT-OSS 关键参数 model_config = { "n_params": 117_000_000_000, # 总参数 "n_active": 5_100_000_000, # 激活参数 "n_layers": 80, # Transformer层数 "n_heads": 96, # 注意力头数 "n_experts": 32, # MoE专家数 "top_k_experts": 2, # 每次选择的专家数 "vocab_size": 200_000, # 词汇表大小 "context_length": 128_000, # 最大上下文长度 "quantization": "MXFP4", # 4位量化 } # 推理流程 def inference(prompt): tokens = tokenize(prompt) embeddings = embed(tokens) + positional_encoding(tokens) for layer in transformer_layers: # Multi-Head Attention attn_output = multi_head_attention(embeddings) embeddings = layer_norm(embeddings + attn_output) # MoE FFN expert_outputs = [] selected_experts = select_top_k_experts(embeddings, k=2) for expert_id in selected_experts: expert_outputs.append(experts[expert_id](embeddings)) ffn_output = weighted_sum(expert_outputs) embeddings = layer_norm(embeddings + ffn_output) logits = output_projection(embeddings) next_token = sample(softmax(logits)) return next_token
